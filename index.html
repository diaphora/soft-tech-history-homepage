History of JavaScript

You may not realize it, but the history of JavaScript is different than the history of Java. In fact, Brendan Eich had originally developed it as a completely different language he called "Mocha," but which he and Netscape changed to JavaScript in December 1995.

JavaScript's original purpose was simple: he designed a language that could be embedded into HTML, allowing web designers and Java programmers to incorporate Java components, like images and plugins onto their websites using JavaScript.

 

Reaction of the Development Community

Given JavaScript's potential for incporating Java components into online content, Ecma International, the standards-development body for information and communication systems, began developing their recommended standards for JavaScript in 1997. In 1999, they released ECMAScript 3, which established the foundations for modern-day JavaScript. Ecma also standardized an Office Open XML format for Microsoft, which has become the default target file format for Microsoft Office.

 

Conflicting Interests

Following the release of ECMAScript 3, both Netscape and Microsoft began implementing proposals for future standards into their browsers. However, though Microsoft implemented some of these early proposals into their Jscript.net language, they later abandoned these efforts to implement core JavaScript features into Internet Explorer.

Development in JavaScript returned from two years of stasis in 2005, when Brendan Eich re-joined Ecma with Mozilla. They began preliminary work on a new standards release alongside developers who had previously worked on Microsoft’s JavaScript research before moving to Macromedia. There they developed ActionScript 3, which became the source for the proposed ECMAScript 4.

However, industrial competition injected conflict into these efforts once more, as both Microsoft and Yahoo joined forces in 2007 to oppose the ECMAScript 4 standards development. Of course, the need to revise and modernize the standards of JavaScript remained, leading to proposals for ECMAScript 3.1, which would include regulations more accommodating to the proprietary languages of major corporations, though these proposals resulted in no major consequences.

 

Enter AJAX

Nonetheless, open-source development communities with an interest in JavaScript arose around libraries like Prototype, jQuery, Dojo, and Mootools. These communities saw the potential for AJAX (Asynchronous JavaScript and XML) to create client-side web applications that load data in the background, freeing the user from having to load an entirely new page, or affecting the display or behavior of an existing page. This was a major development in both Internet technologies, and signaled the advent of dynamic web content.

As the broader development community discovered the value of AJAX, the campaign to develop revised ECMAScript standards saw a revival. In 2008, developers met in Oslo to discuss ECMAScript 3.1 in terms of newer AJAX applications, which proved revolutionary enough in terms of standardization requirements that developers renamed it ECMAScript 5.

Following the release of the ECMAScript 5 standards, JavaScript has seen more server-side developments using the newly-developed Node.js. Besides that, JavaScript can now use HTML5 in order to control user media, like re-sizing video for mobile displays.
